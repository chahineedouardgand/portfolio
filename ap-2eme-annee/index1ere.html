1
Objectif du projet

Ce projet de 1ère année a pour objectif de créer et configurer un serveur Windows Server complet dans un environnement virtualisé. Il couvre l'installation du système, la configuration réseau, le déploiement d'Active Directory et la gestion des utilisateurs — compétences fondamentales pour un futur administrateur systèmes et réseaux.

2
Étape 1 — Création de la VM avec VirtualBox

La première étape consiste à créer une machine virtuelle sous Windows Server à l'aide d'Oracle VirtualBox. La VM est configurée avec les ressources nécessaires, puis le réseau est mis en mode Accès par pont pour que la VM puisse communiquer avec le réseau local du lycée.

3
Étape 2 — Installation du rôle AD DS

Une fois Windows Server installé et à jour, le rôle AD DS (Active Directory Domain Services) est ajouté via le Gestionnaire de serveur. Ce rôle permet de transformer le serveur en contrôleur de domaine — le cœur de la gestion centralisée des utilisateurs et des machines dans un réseau d'entreprise.

  • Ouvrir le Gestionnaire de serveur → Gérer → Ajouter des rôles et fonctionnalités
  • Sélectionner Services AD DS dans la liste des rôles → Installer
  • Après installation → cliquer sur la notification "Promouvoir ce serveur en contrôleur de domaine"
  • Choisir Ajouter une nouvelle forêt → saisir le nom de domaine (ex : mondomaine.local)
  • Définir le mot de passe DSRM → laisser les options par défaut → Installer → Redémarrage automatique
4
Étape 3 — Vérification du contrôleur de domaine

Après le redémarrage, on vérifie que le serveur fonctionne correctement en tant que contrôleur de domaine à l'aide des commandes de diagnostic :

PowerShell / CMD — Vérifications REM Vérifier la configuration réseau
ipconfig /all

REM Diagnostiquer l'Active Directory
dcdiag

REM Résultat attendu : tous les tests = Passed
REM "DC: TEST: Advertising ... passed"

REM Vérifier les services AD
net logon
net start | findstr Active

6
Compétences SISR développées
  • Virtualisation — Créer et configurer une VM avec VirtualBox, gérer les ressources et le réseau
  • Administration Windows — Installer et configurer Windows Server, gérer les rôles et fonctionnalités
  • Active Directory — Déployer un contrôleur de domaine, créer des utilisateurs et des OUs
  • Politique de groupe — Comprendre et appliquer des GPO pour centraliser la configuration des postes
  • Diagnostic — Utiliser dcdiag, ipconfig et les journaux d'événements pour vérifier le bon fonctionnement
  • Travail en mode projet — Planifier les étapes, travailler en autonomie, vérifier à chaque phase
Résultat obtenu

La VM Windows Server est opérationnelle avec Active Directory entièrement configuré. Le serveur fonctionne en contrôleur de domaine, vérifié par dcdiag (tous les tests passés) et ipconfig. Les utilisateurs, les groupes et les OUs sont créés. Les GPO sont appliquées et mises à jour via gpupdate /force.
