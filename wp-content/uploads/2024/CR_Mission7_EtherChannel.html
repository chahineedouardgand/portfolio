Principe de l'EtherChannel
Qu'est-ce que l'EtherChannel ?

L'EtherChannel regroupe plusieurs interfaces physiques entre deux switches pour créer un seul lien logique appelé Port-Channel.

Du point de vue du réseau, ce lien logique est traité comme une seule interface — le Spanning Tree Protocol (STP) ne bloque donc pas les ports redondants.

Ici : Fa0/1 + Fa0/2 → Port-Channel 1 (Po1) — bande passante cumulée : 2 × 100 Mbit/s = 200 Mbit/s.

Avantages de l'EtherChannel
⚡
Bande passante augmentée
2 liens de 100 Mbit/s cumulés = 200 Mbit/s de capacité totale entre les deux switches. Idéal pour les liens inter-switches à fort trafic.
🛡️
Tolérance aux pannes
Si Fa0/1 tombe en panne, Fa0/2 continue de transporter le trafic automatiquement — aucune interruption de service.
⚖️
Répartition de charge
Le trafic est réparti automatiquement entre les liens physiques selon un algorithme (source/destination MAC ou IP).
🗺️
Topologie simplifiée
STP voit un seul lien logique au lieu de 2 — pas de port bloqué, pas de boucle. Infrastructure plus robuste et plus facile à gérer.
3
Protocoles d'agrégation — LACP vs PAgP
LACP — Link Aggregation Control Protocol
✅ Standard IEEE 802.3ad — Recommandé

Protocole ouvert et standard — compatible avec tous les équipements réseau (Cisco, HP, Juniper…).

Utilisé dans ce TP car le réseau CRASH peut inclure des équipements de marques différentes.

active
→ Envoie activement des PDU LACP — initie la négociation
passive
→ Attend les PDU LACP — répond à la négociation
PAgP — Port Aggregation Protocol
⚠️ Propriétaire Cisco uniquement

Protocole propriétaire Cisco — fonctionne uniquement entre équipements Cisco.

Non utilisé ici car moins interopérable, mais syntaxe similaire.

desirable
→ Équivalent de "active" pour PAgP
auto
→ Équivalent de "passive" pour PAgP

Câblage physique entre les deux commutateurs
Pourquoi le câblage physique est-il la première étape indispensable ?

EtherChannel ne fonctionne que si les ports des deux switches sont connectés ensemble — les ports Fa0/1 des deux switches entre eux, idem pour Fa0/2.

Si les câbles sont croisés (Fa0/1 Switch A → Fa0/2 Switch B), l'agrégation ne pourra pas se former.

Avant la configuration logicielle, la topologie physique doit être vérifiée.

Figure 1 — Double liaison Ethernet entre les deux switches
Câblage physique requis
! Connexions physiques à réaliser AVANT la configuration :

Switch A  Fa0/1  ←————————→  Fa0/1  Switch B
Switch A  Fa0/2  ←————————→  Fa0/2  Switch B

! Vérifier que les LEDs des ports sont allumées (lien physique actif)
! Commande de vérification du lien physique :
Switch# show interfaces fa0/1
FastEthernet0/1 is up, line protocol is up
Switch# show interfaces fa0/2
FastEthernet0/2 is up, line protocol is up
⌨ Vérification physique
show interfaces fa0/1 / show interfaces fa0/2
🧠 Pourquoi vérifier avant de configurer ?
Si un lien physique est down, EtherChannel refusera de se former ou sera incomplet. is up, line protocol is up confirme que le câble est branché et que le lien Ethernet est opérationnel des deux côtés.
✅ Résultat attendu
Les deux ports Fa0/1 et Fa0/2 sont physiquement connectés et actifs — la configuration EtherChannel peut commencer.

Configuration EtherChannel — LACP mode active
Règle importante : même configuration sur les deux switches

Les commandes ci-dessous doivent être identiques sur Switch A et Switch B.

Si un côté est en mode active et l'autre en passive → EtherChannel se forme. Si les deux sont en passive → aucun ne prend l'initiative → pas d'agrégation.

Capture 1 — Configuration sur Switch A (identique sur Switch B)
Switch Cisco — EtherChannel LACP
! Entrer en mode configuration globale
Switch> enable
Switch# configure terminal

! Sélectionner les deux ports à agréger en même temps
Switch(config)# interface range fa0/1 - 2

! Créer le Port-Channel 1 avec le protocole LACP en mode active
Switch(config-if-range)# channel-group 1 mode active

! S'assurer que les ports ne sont pas désactivés
Switch(config-if-range)# no shutdown
Switch(config-if-range)# exit

! Sauvegarder la configuration — OBLIGATOIRE sur matériel réel
Switch# write memory

! Faire exactement la même configuration sur Switch B
interface range fa0/1 - 2
Pourquoi ? Sélectionne simultanément les interfaces Fa0/1 et Fa0/2. Toutes les commandes suivantes s'appliquent aux deux ports en même temps — gain de temps et garantie de cohérence de configuration.
channel-group 1 mode active
Pourquoi ? Crée le Port-Channel 1 (Po1) et active le protocole LACP en mode active → ce switch envoie des PDU LACP pour négocier l'agrégation avec l'autre switch. Le Port-Channel 1 apparaît automatiquement dans la liste des interfaces.
no shutdown
Pourquoi ? S'assure que les ports physiques sont bien activés. Un port en shutdown ne peut pas participer à un EtherChannel.
⌨ Commandes clés à retenir
interface range fa0/1 - 2 + channel-group 1 mode active + no shutdown
🧠 Pourquoi "mode active" et pas "mode passive" sur les deux ?
LACP nécessite qu'au moins un des deux switches soit en mode active pour initier la négociation. Si les deux sont en passive, personne n'envoie de PDU → pas d'agrégation. Mettre les deux en active fonctionne aussi et est plus simple.
✅ Résultat attendu
Le Port-Channel Po1 se forme automatiquement — Fa0/1 et Fa0/2 sont agrégés en un seul lien logique de 200 Mbit/s.

write memory → toujours sauvegarder sur les deux switches — la config est perdue au redémarrage sinon.

Symétrie obligatoire → la configuration doit être identique sur les deux switches (même numéro de channel-group, même mode, mêmes ports).

active + passive → combinaison valide. passive + passive → EtherChannel ne se forme pas (personne n'initie).
