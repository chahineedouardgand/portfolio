Objectif de la mission

Cette mission a pour objectif de segmenter le réseau interne du projet CRASH à l'aide de VLAN, de centraliser leur gestion grâce au protocole VTP, et de mettre en place un routage Inter-VLAN via le routeur Cisco 2901. Cette architecture est utilisée en entreprise pour améliorer la sécurité, les performances et l'organisation du réseau.

Segmentation VLAN

Créer des réseaux logiques isolés (Escrime / Volley) sur le même switch physique.

Gestion VTP

Synchroniser automatiquement les VLAN entre tous les switches du réseau.

Routage Inter-VLAN

Faire communiquer les VLAN grâce à la méthode Router-on-a-Stick sur le Cisco 2901.

Création des VLAN sur le switch
Qu'est-ce qu'un VLAN ?

Un VLAN (Virtual LAN) crée des réseaux logiques séparés sur le même switch physique.

VLAN 10 = Ligue Escrime | VLAN 20 = Ligue Volley → chaque ligue est isolée du reste.

Figure 1 — Commandes de création des VLAN
Switch Cisco — Création VLAN
! Entrer en mode privilégié puis configuration globale
Switch> enable
Switch# configure terminal

! Créer le VLAN 10 et lui donner un nom
Switch(config)# vlan 10
Switch(config-vlan)# name Escrime
Switch(config-vlan)# exit

! Créer le VLAN 20 et lui donner un nom
Switch(config)# vlan 20
Switch(config-vlan)# name Volley
Switch(config-vlan)# exit
Switch# write memory     ! Toujours sauvegarder !
⌨ Commande(s) utilisée(s)
vlan 10 / vlan 20 / name <nom>
🧠 Pourquoi cette commande ?
La commande vlan crée le VLAN dans la base de données du switch.
name lui donne un nom lisible pour identifier facilement le service auquel il appartient.
✅ Résultat attendu
Deux réseaux logiques isolés (Escrime / Volley) créés sur le même switch physique.
Figure 2 — Vérification des VLAN (show vlan brief)
Vérification
! Vérifier que les VLAN sont bien créés et actifs
Switch# show vlan brief

! Résultat attendu dans la capture d'écran :
VLAN  Name         Status    Ports
10    Escrime      active
20    Volley       active
⌨ Commande(s) utilisée(s)
show vlan brief
🧠 Pourquoi cette commande ?
Liste tous les VLAN existants avec leur statut. Si un VLAN n'apparaît pas ici → il n'existe pas et aucun trafic ne l'utilisera.
✅ Résultat attendu
VLAN 10 (Escrime) et VLAN 20 (Volley) apparaissent en statut active.

Mise en place du protocole VTP
Pourquoi VTP ?

VTP (VLAN Trunking Protocol) synchronise automatiquement les VLAN entre tous les switches.

Sans VTP → il faut créer chaque VLAN manuellement sur chaque switch du réseau.

Avec VTP mode Server → un seul switch centralise et diffuse la liste des VLAN à tous les autres.

Figure 3 — Configuration du domaine VTP — mode serveur
VTP — Switch Serveur
! Ce switch sera le serveur VTP (il diffuse les VLAN)
Switch(config)# vtp mode server

! Définir le domaine VTP — MÊME nom sur tous les switches !
Switch(config)# vtp domain CRASH

! Définir la version VTP (recommandé : version 2)
Switch(config)# vtp version 2
Switch# write memory
⌨ Commande(s) utilisée(s)
vtp mode server / vtp domain CRASH
🧠 Pourquoi cette commande ?
vtp mode server → ce switch est l'autorité qui diffuse la liste des VLAN.
vtp domain CRASH → tous les switches configurés avec ce même domaine reçoivent automatiquement les VLAN.
Les switches en mode client appliquent les changements sans configuration manuelle.
✅ Résultat attendu
Un seul switch à configurer pour propager les VLAN sur tout le réseau.
Figure 4 — Vérification du statut VTP
Vérification VTP
! Vérifier le mode et le domaine VTP
Switch# show vtp status

! Résultat attendu dans la capture d'écran :
VTP Operating Mode      : Server
VTP Domain Name         : CRASH
Number of existing VLANs: 2
⌨ Commande(s) utilisée(s)
show vtp status
🧠 Pourquoi cette commande ?
Confirme que le switch est bien en mode Server dans le bon domaine. Si Number of existing VLANs = 2 → les VLAN 10 et 20 sont correctement synchronisés.
✅ Résultat attendu
Le switch diffuse correctement VLAN 10 et VLAN 20 sur tout le réseau.

Configuration du port Trunk vers le routeur
Pourquoi un port trunk est-il indispensable ?

Un port trunk transporte plusieurs VLAN sur un seul câble physique grâce à l'encapsulation 802.1Q.

Sans trunk → le routeur ne verrait qu'un seul VLAN → routage Inter-VLAN impossible.

Chaque trame est taguée avec son numéro de VLAN → le routeur sait à quel VLAN elle appartient.

Figure 5 — Activation du mode trunk — GigabitEthernet0/1
Switch — Trunk 802.1Q
! Sélectionner le port physique connecté au routeur
Switch(config)# interface GigabitEthernet0/1

! Autoriser uniquement les VLAN 10 et 20 à transiter
Switch(config-if)# switchport trunk allowed vlan 10,20

! Activer le mode trunk (encapsulation 802.1Q)
Switch(config-if)# switchport mode trunk
Switch(config-if)# no shutdown
Switch(config-if)# exit
Switch# write memory     ! CRUCIAL sur matériel réel !
⌨ Commande(s) utilisée(s)
switchport trunk allowed vlan 10,20 + switchport mode trunk
🧠 Pourquoi cette commande ?
switchport trunk allowed vlan 10,20 → seuls ces VLAN passent sur le câble, les autres sont bloqués.
switchport mode trunk → active le tag 802.1Q : chaque trame est marquée avec son numéro de VLAN.
write memory → CRUCIAL sur matériel réel : sinon la config est perdue au redémarrage !
✅ Résultat attendu
Les VLAN 10 et 20 peuvent maintenant circuler physiquement vers le routeur Cisco 2901.
Figure 6 — Vérification du port trunk
Vérification Trunk
! Vérifier que le port est bien en mode trunk
Switch# show interfaces trunk

! Résultat attendu dans la capture d'écran :
Port     Mode    Encapsulation  Status
Gi0/1    on      802.1q         trunking

VLANs allowed and active: 10, 20
⌨ Commande(s) utilisée(s)
show interfaces trunk
🧠 Pourquoi cette commande ?
Confirme que le port est en mode trunking avec l'encapsulation 802.1q.
"VLANs allowed and active: 10, 20" prouve que les deux VLAN transitent bien sur ce câble.
✅ Résultat attendu
Le trunk est opérationnel — le routeur peut recevoir et distinguer les VLAN 10 et 20.

Routage Inter-VLAN — Router-on-a-Stick
Qu'est-ce que le Router-on-a-Stick ?

Par défaut, deux VLAN différents NE peuvent PAS communiquer entre eux.

Solution : créer des sous-interfaces sur le routeur — une par VLAN — sur un seul câble physique.

Gi0/1.10 → passerelle VLAN 10 (172.17.10.1)  |  Gi0/1.20 → passerelle VLAN 20 (172.17.20.1)

Le routeur lit le tag 802.1Q et sait vers quel sous-réseau router chaque paquet.

Figure 7 — Sous-interfaces VLAN sur le routeur Cisco 2901
Routeur Cisco 2901
! Activer l'interface physique principale (sans IP directe)
Router(config)# interface GigabitEthernet0/1
Router(config-if)# no shutdown
Router(config-if)# exit

! Sous-interface VLAN 10 — passerelle des machines Escrime
Router(config)# interface GigabitEthernet0/1.10
Router(config-subif)# encapsulation dot1Q 10
Router(config-subif)# ip address 172.17.10.1 255.255.255.0
Router(config-subif)# no shutdown

! Sous-interface VLAN 20 — passerelle des machines Volley
Router(config)# interface GigabitEthernet0/1.20
Router(config-subif)# encapsulation dot1Q 20
Router(config-subif)# ip address 172.17.20.1 255.255.255.0
Router(config-subif)# no shutdown
Router# write memory     ! Sauvegarder la configuration !
⌨ Commande(s) utilisée(s)
interface Gi0/1.10 + encapsulation dot1Q 10 + ip address 172.17.10.1
🧠 Pourquoi cette commande ?
interface Gi0/1.10 → crée une sous-interface logique dédiée au VLAN 10.
encapsulation dot1Q 10 → les trames taguées VLAN 10 arrivent sur cette interface.
ip address 172.17.10.1 → c'est la passerelle par défaut des machines du VLAN 10.
Même principe pour le VLAN 20 avec l'IP 172.17.20.1.
✅ Résultat attendu
Le routeur connaît les 2 sous-réseaux et peut router les paquets de l'un vers l'autre.
Figure 8 — Vérification des interfaces du routeur
Vérification Routeur
! Vérifier que les sous-interfaces sont actives
Router# show ip interface brief

! Résultat attendu dans la capture d'écran :
Interface              IP-Address     OK? Status   Protocol
GigabitEthernet0/1     unassigned     YES up        up
GigabitEthernet0/1.10  172.17.10.1    YES up        up
GigabitEthernet0/1.20  172.17.20.1    YES up        up
⌨ Commande(s) utilisée(s)
show ip interface brief
🧠 Pourquoi cette commande ?
Affiche toutes les interfaces avec leur IP et leur statut. Si Gi0/1.10 et Gi0/1.20 sont up/up avec leurs IP → le routage Inter-VLAN est actif.
✅ Résultat attendu
Les deux sous-interfaces sont actives → VLAN 10 et VLAN 20 peuvent maintenant communiquer.
