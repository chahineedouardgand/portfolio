Pourquoi mettre en place le Port-Security ?
Le problème sans Port-Security

N'importe qui peut brancher un PC non autorisé, un hub, ou lancer une attaque MAC flooding pour saturer la table CAM du switch.

Une attaque MAC flooding remplit la table des adresses MAC → le switch se comporte comme un hub → tout le trafic est diffusé à tous les ports → interception possible.

🚫 Branchements sauvages

Empêcher les PC non autorisés de se connecter au réseau sans permission.

⚡ MAC Flooding

Bloquer les attaques qui saturent la table CAM pour intercepter le trafic réseau.

🔒 1 MAC par port

Dans le projet CRASH, chaque port utilisateur est limité à une seule adresse MAC autorisée.

Principe : si une autre machine est branchée sur le port, celui-ci est automatiquement bloqué (err-disabled). Cette configuration est essentielle en entreprise pour garantir la sécurité physique du réseau.

Configuration du Port-Security
Comment fonctionne le mode Sticky ?

Le mode sticky permet au switch d'apprendre automatiquement la première adresse MAC branchée sur le port.

Cette MAC est ensuite mémorisée dans la configuration en cours — pas besoin de la saisir manuellement.

Si une MAC différente tente de se connecter → violation détectée → port coupé automatiquement.

Figure 1 — Configuration complète du Port-Security sur Fa0/3
Switch Cisco — Port Security
! Entrer en mode configuration sur le port Fa0/3
Switch> enable
Switch# configure terminal
Switch(config)# interface fastEthernet0/3

! Forcer le port en mode access (obligatoire pour Port-Security)
Switch(config-if)# switchport mode access

! Activer le Port-Security sur ce port
Switch(config-if)# switchport port-security

! Autoriser une seule adresse MAC sur ce port
Switch(config-if)# switchport port-security maximum 1

! Apprentissage automatique de la MAC branchée (mode sticky)
Switch(config-if)# switchport port-security mac-address sticky

! En cas de violation : couper le port (err-disabled)
Switch(config-if)# switchport port-security violation shutdown

Switch(config-if)# exit
Switch# write memory     ! Sauvegarder — CRUCIAL sur matériel réel !
switchport mode access
Pourquoi ? Le Port-Security ne fonctionne QUE sur les ports en mode access. Un port trunk ne peut pas être sécurisé par cette méthode.
switchport port-security
Pourquoi ? Active la fonctionnalité Port-Security sur ce port. Sans cette ligne, toutes les autres commandes port-security sont ignorées.
maximum 1
Pourquoi ? Limite à une seule adresse MAC autorisée. Si un hub ou un switch non autorisé est branché (plusieurs MACs), violation immédiate.
mac-address sticky
Pourquoi ? Le switch apprend automatiquement la première MAC branchée et la mémorise. Évite de saisir manuellement l'adresse MAC de chaque machine.
violation shutdown
Pourquoi ? En cas de violation détectée (MAC non autorisée), le port passe en état err-disabled — coupure totale, aucune communication n'est possible.
⌨ Commandes clés à retenir
switchport port-security + maximum 1 + mac-address sticky + violation shutdown
🧠 Pourquoi cette configuration ?
Ces 4 commandes ensemble forment un bouclier : le port n'accepte qu'une seule machine (celle qui a été branchée en premier). Toute autre machine est automatiquement rejetée et le port est coupé.
✅ Résultat attendu
Le port Fa0/3 est sécurisé — il apprend automatiquement la MAC de la machine autorisée et bloque toute autre connexion.

Figure 3 — Détection de violation de sécurité
Log switch — Violation détectée
! Message affiché dans la console du switch lors de la violation :
%PORT_SECURITY-2-PSECURE_VIOLATION: Security violation occurred,
caused by MAC address 00bb.ccdd.eeff on port FastEthernet0/3.

! Le port passe automatiquement en err-disabled :
%LINEPROTO-5-UPDOWN: Line protocol on Interface FastEthernet0/3,
changed state to down
%LINK-3-UPDOWN: Interface FastEthernet0/3, changed state to down
⌨ Ce qui se passe lors de la violation
violation shutdown → port passe en err-disabled
🧠 Pourquoi ce comportement ?
Le mode violation shutdown est le plus sécurisé : il coupe totalement le port dès qu'une MAC non autorisée est détectée. Le switch envoie un log d'alerte et incrémente le compteur de violations.
✅ Résultat attendu
Le port Fa0/3 passe en état err-disabled — aucune communication n'est possible depuis ce port.

Vérification de l'état du port bloqué
Figure 4 — Port bloqué en err-disabled
Vérification état port
! Vérifier l'état du port après violation
Switch# show port-security interface fastEthernet0/3

! Résultat attendu dans la capture d'écran :
Port Security              : Enabled
Port Status                : Secure-shutdown
Violation Mode             : Shutdown
Security Violation Count   : 1

! Vérifier aussi dans la table des interfaces :
Switch# show interfaces fastEthernet0/3 status

! Résultat :
Port     Name    Status          Vlan
Fa0/3            err-disabled    1
⌨ Commande(s) utilisée(s)
show port-security interface fa0/3 / show interfaces fa0/3 status
🧠 Pourquoi ?
Port Status: Secure-shutdown confirme que la violation a bien été détectée et que le port est bloqué. Security Violation Count: 1 indique qu'une intrusion a eu lieu — tracé pour l'audit.
✅ Résultat attendu
Le port apparaît en err-disabled — la sécurité fonctionne correctement, aucun trafic ne peut transiter.
7
Procédure de remise en service du port
Quand utiliser cette procédure ?

Uniquement après avoir vérifié et rebranché l'équipement autorisé sur le port. Ne jamais réactiver sans contrôle préalable.

Figure 5 — Remise en service du port Fa0/3
Réactivation port
! Entrer sur l'interface bloquée
Switch(config)# interface fastEthernet0/3

! Éteindre manuellement le port (reset de l'état err-disabled)
Switch(config-if)# shutdown

! Réactiver le port
Switch(config-if)# no shutdown

Switch(config-if)# exit
Switch# write memory

! Vérification après remise en service :
Switch# show port-security interface fastEthernet0/3
Port Status                : Secure-up
Security Violation Count   : 0
⌨ Commande(s) utilisée(s)
shutdown puis no shutdown
🧠 Pourquoi shutdown puis no shutdown ?
Un port en err-disabled ne revient pas tout seul en service. Il faut le forcer manuellement avec shutdown pour réinitialiser l'état, puis no shutdown pour le réactiver. À utiliser uniquement après avoir vérifié l'équipement branché.
✅ Résultat attendu
Le port repasse en état Secure-up — il est de nouveau opérationnel avec la sécurité toujours active.
