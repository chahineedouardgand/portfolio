Contexte et objectif de la mission
Pourquoi les ACL sont-elles indispensables sur un routeur ?

Le routeur est un point critique du réseau : il assure la communication entre le LAN, la DMZ et Internet.

Sans filtrage, n'importe quel trafic peut entrer ou sortir — brèche de sécurité majeure.

Les ACL permettent de contrôler précisément chaque flux en autorisant ou bloquant selon des règles définies.

Cette mission configure deux ACL : ACL 100 (protection LAN/DMZ) et ACL 101 (sécurisation Internet). Les flux bloqués sont journalisés dans Syslog pour traçabilité.

Principe de fonctionnement des ACL
Comment fonctionne une ACL Cisco ?

Une ACL est une liste de règles lues de haut en bas : dès qu'une règle correspond au trafic, elle est appliquée — les suivantes sont ignorées.

Chaque règle peut : permit (autoriser) ou deny (bloquer) le trafic.

Règle implicite finale : à la fin de toute ACL Cisco, une règle invisible deny ip any any bloque tout ce qui n'a pas été explicitement autorisé.

🧠 Pourquoi activer les logs avant les ACL ?
Sans horodatage, les logs sont illisibles — impossible de savoir quand une tentative de connexion a eu lieu. logging buffered stocke les événements en mémoire pour analyse ultérieure avec show logging.

ACL 100 — Protection LAN et DMZ
Objectifs de l'ACL 100

Contrôler ce qui entre dans le routeur depuis le LAN et la DMZ — seul le trafic interne autorisé peut traverser le routeur.

Tout trafic non reconnu (ex : une machine pirate avec une IP externe) sera bloqué et journalisé.

PERMIT
access-list 100 permit ip 172.17.0.0 0.0.255.255 any
→ Autorise tout le trafic IP provenant du LAN (172.17.0.0/16) vers n'importe quelle destination
PERMIT
access-list 100 permit ip 10.54.0.0 0.0.0.255 any
→ Autorise tout le trafic IP provenant de la DMZ (10.54.0.0/24) vers n'importe quelle destination
DENY + LOG
access-list 100 deny ip any any log
→ Bloque TOUT autre trafic et l'enregistre dans les journaux Syslog pour traçabilité
Capture 2 — Création de l'ACL 100
Routeur CRASH — ACL 100
Router(config)# conf t

! Règle 1 : Autoriser le LAN (wildcard 0.0.255.255 = masque inversé de /16)
Router(config)# access-list 100 permit ip 172.17.0.0 0.0.255.255 any

! Règle 2 : Autoriser la DMZ (wildcard 0.0.0.255 = masque inversé de /24)
Router(config)# access-list 100 permit ip 10.54.0.0 0.0.0.255 any

! Règle 3 : Bloquer tout le reste + log dans Syslog
Router(config)# access-list 100 deny ip any any log
⌨ Notion clé : le wildcard mask
0.0.255.255 = masque inversé de /16 | 0.0.0.255 = masque inversé de /24
🧠 Pourquoi un wildcard et non un masque normal ?
Cisco IOS utilise des wildcards (masques inversés) dans les ACL : 0 = bit fixe (doit correspondre), 1 = bit libre (peu importe). 0.0.255.255 signifie "seuls les 16 premiers bits comptent" → correspond à tout le réseau 172.17.0.0/16.
✅ Résultat attendu
Seul le trafic du LAN et de la DMZ est autorisé à entrer dans le routeur — toute autre source est bloquée et tracée.
Capture 3 — Application de l'ACL 100 sur les interfaces LAN et DMZ
Application sur interfaces
Router(config)# conf t

! Appliquer ACL 100 en entrée sur l'interface LAN (G0/0)
Router(config)# interface g0/0
Router(config-if)# ip access-group 100 in
Router(config-if)# exit

! Appliquer ACL 100 en entrée sur l'interface DMZ (G0/1)
Router(config)# interface g0/1
Router(config-if)# ip access-group 100 in
Router(config-if)# exit

Router# write memory     ! Sauvegarder obligatoirement
⌨ Commande(s) utilisée(s)
ip access-group 100 in
🧠 Pourquoi "in" et pas "out" ?
in → filtre le trafic qui entre dans l'interface (depuis le LAN/DMZ vers le routeur). C'est le sens correct : on filtre à l'entrée pour bloquer avant même que le routeur ne traite le paquet. out filtrerait à la sortie, ce qui serait trop tardif.
✅ Résultat attendu
L'ACL 100 est active sur G0/0 (LAN) et G0/1 (DMZ) — tout trafic entrant depuis ces interfaces est filtré selon les règles définies.

ACL 101 — Sécurisation des flux Internet
Objectifs de l'ACL 101

Contrôler les flux entre le réseau interne, la DMZ et Internet — autoriser uniquement ce qui est légitime.

Bloquer tout accès entrant depuis Internet vers le LAN — protéger le réseau interne d'intrusions.

PERMIT
access-list 101 permit tcp 172.17.0.0 0.0.255.255 host 192.168.230.1 eq 80
→ Autorise la navigation HTTP (port 80) du LAN vers Internet
PERMIT
access-list 101 permit tcp 172.17.0.0 0.0.255.255 host 192.168.230.1 eq 443
→ Autorise la navigation HTTPS (port 443) du LAN vers Internet
PERMIT
access-list 101 permit icmp 172.17.0.0 0.0.255.255 host 192.168.230.1 echo
→ Autorise le ping sortant du LAN vers Internet
PERMIT
access-list 101 permit icmp 10.54.0.0 0.0.0.255 host 192.168.230.1 echo
→ Autorise le ping sortant de la DMZ vers Internet
DENY + LOG
access-list 101 deny ip any any log
→ Bloque TOUT le reste (ping entrant depuis Internet, accès non autorisés) + journalisation
Capture 4 — Création de l'ACL 101
Routeur CRASH — ACL 101
Router(config)# conf t

! Autoriser HTTP et HTTPS du LAN vers Internet
Router(config)# access-list 101 permit tcp 172.17.0.0 0.0.255.255 host 192.168.230.1 eq 80
Router(config)# access-list 101 permit tcp 172.17.0.0 0.0.255.255 host 192.168.230.1 eq 443

! Autoriser le ping sortant (LAN → Internet et DMZ → Internet)
Router(config)# access-list 101 permit icmp 172.17.0.0 0.0.255.255 host 192.168.230.1 echo
Router(config)# access-list 101 permit icmp 10.54.0.0 0.0.0.255 host 192.168.230.1 echo

! Bloquer tout le reste + log (ping entrant Internet bloqué, accès non autorisés)
Router(config)# access-list 101 deny ip any any log
⌨ Mots-clés importants
eq 80 / eq 443 = ports HTTP/HTTPS | echo = requête ping | host = IP unique
🧠 Pourquoi filtrer par port (eq 80, eq 443) ?
eq 80 / eq 443 → filtre uniquement les connexions Web — les autres ports (FTP, Telnet, SSH depuis Internet) restent bloqués. echo → autorise uniquement les requêtes ping sortantes, pas les réponses entrantes d'Internet non sollicitées.
✅ Résultat attendu
Le LAN peut naviguer sur Internet (HTTP/HTTPS) et pinger — mais Internet ne peut pas accéder au LAN ou pinger de l'intérieur.
Capture 5 — Application de l'ACL 101 sur l'interface Internet (G0/2)
Application — Interface Internet
Router(config)# interface g0/2

! Filtrer le trafic entrant depuis Internet
Router(config-if)# ip access-group 101 in

! Filtrer le trafic sortant vers Internet
Router(config-if)# ip access-group 101 out

Router(config-if)# exit
Router# write memory
⌨ Commande(s) utilisée(s)
ip access-group 101 in + ip access-group 101 out
🧠 Pourquoi appliquer dans les deux sens ici ?
in → filtre le trafic venant d'Internet (bloque les connexions entrantes non autorisées). out → filtre le trafic sortant vers Internet (garantit que seul le trafic autorisé peut quitter le réseau). Double filtrage = sécurité renforcée.
✅ Résultat attendu
L'ACL 101 contrôle les flux dans les deux sens sur l'interface Internet — protection complète des deux côtés.

🧠 Pourquoi vérifier les compteurs ACL ?
show ip access-lists affiche le nombre de paquets traités par chaque règle (matches). Si une règle a 0 match → soit elle ne sert à rien, soit il y a une erreur de configuration. show logging montre les tentatives bloquées avec horodatage — permet de détecter des attaques ou des erreurs de configuration.
✅ Résultat attendu
Les règles permit ont des matches → le trafic autorisé passe. Les règles deny ont des matches → les tentatives bloquées sont enregistrées.
